# XDFs and Definitions

> **Source confidence:** Needs validation

## What it is

An XDF (or similar definition file) describes where each calibration map lives inside a ROM and how to interpret its raw bytes as tables, scalars, and axes.

## Why it matters

Without a matched definition, a BIN is just bytes. The definition turns it into readable maps — and it must match the exact ROM ID.

## How it works

The definition stores addresses, sizes, and math (scaling) for each parameter. Tuning software uses it to display maps.

## Common symptoms or failure modes

- Wrong/garbage values shown when a definition does not match the ROM

## Mechanical checks first

Confirm definition-to-ROM match before trusting any displayed map. This wiki does not distribute private XDFs.
