# WGDC Correction Map

> **Source confidence:** Needs validation

> Educational reference only. This page does not publish tune values.

## What this map controls

The WGDC correction (closed-loop) logic trims duty cycle to drive actual boost toward target. It corrects for conditions the base map cannot anticipate.

## Why it matters

Aggressive correction can oscillate; weak correction can leave persistent error. It must be tuned against real logs.

## Related log channels

- WGDC
- Boost target vs actual

## What changes are visible in logs

Changes show as different correction behavior and how quickly/cleanly boost converges to target.

## What can go wrong if changed blindly

- Oscillation/hunting
- Slow convergence
- Masking a mechanical leak

## Mechanical problems that can look like a tuning problem

- Boost leaks
- Failing solenoids/wastegates
