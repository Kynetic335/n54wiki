# N54 UI UX Design Rules

Design for technical learning, scanning, and comparison.

## Principles

- Keep the interface quiet, precise, and content-forward.
- Preserve existing route, layout, component, and styling conventions.
- Use dense but readable technical layouts for lessons, parameters, diagnostics, and guides.
- Prioritize tables, callouts, checklists, and structured sections over decorative marketing blocks.
- Keep cards restrained and avoid nesting cards inside cards.

## Public Wiki UI Boundaries

- Do not add UI for flashing, BIN writing, tune generation, checksum correction, private file upload, private file browsing, or private downloads.
- Do not expose protected file names, raw calibration snippets, customer identifiers, or private source inventories.
- Do not add route links that imply private tooling exists in the public app.

## Accessibility And Responsiveness

- Keep text readable on mobile and desktop.
- Avoid overlapping controls or text.
- Use semantic headings and link text.
- Preserve keyboard-accessible controls for interactive elements.
